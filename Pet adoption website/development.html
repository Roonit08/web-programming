Development & Compliance

This page explains how the website was planned, designed, developed, and tested. It also covers privacy, security, copyright, and content appropriateness.

Project overview

The goal of this project was to create a simple, welcoming website for a local animal rescue called “Happy Paws Pet Adoption”. The site is designed for first-time visitors who want to learn about the organisation, view available pets, understand the adoption process, and send a message through a contact form.

Features

  • Consistent navigation menu across all pages
  • Full-screen hero section on the Home page with a background image
  • Responsive layout (works on desktop, tablet, and mobile)
  • Available Pets page with simple “pet cards” (image, name, age, short description)
  • Contact form that redirects to a Thank You page after submission
  • Privacy notice on the Contact page

Technologies used

  • HTML5 (page structure and content)
  • CSS3 (Bondi Blue colour palette, layout, spacing, responsive design)
  • JavaScript (small script for mobile navigation toggle)

How the project works

This is a static website, so there is no real database or server-side form handling. The contact form uses a simple HTML action that loads thankyou.html after the user submits the message. This matches the assignment requirement to show a Thank You page.

Design decisions

I chose a Bondi Blue theme because it feels friendly, clean, and fits a community rescue vibe. The layout is intentionally simple: clear headings, generous spacing, and consistent card sections so the content is easy to scan.

For responsiveness, I used flexible grids and a mobile menu toggle. I avoided complicated frameworks because the goal was clean, beginner-friendly code.

Challenges faced

  • Keeping the design modern without making the code too complex
  • Making the navigation work nicely on smaller screens
  • Ensuring images looked consistent across different screen sizes

Conclusion

Overall, the website meets the scenario requirements and stays simple enough to understand and maintain. If this were a real rescue site, the next step would be adding a backend for real pet listings and secure form handling.

Testing summary

Before final submission, I tested the website manually using different screen sizes (desktop, tablet, and mobile view in browser dev tools).

  • Page links: Clicked every navigation link on every page to confirm they work.
  • Navigation: Checked that the menu collapses on smaller screens and expands using the Menu button.
  • Images: Verified all image paths are correct and images load; added alt text for accessibility.
  • Form submission behaviour: Submitted the form on Contact Us and confirmed it loads thankyou.html and the Home button works.

Privacy, security, copyright, and content appropriateness

Privacy issue

The contact form collects personal information (name and email). Even for a simple enquiry form, users should know why the data is being collected. I included a short privacy notice on the Contact Us page explaining the purpose.

Security issue

A real website must protect user data and prevent misuse (for example, spam submissions or storing data insecurely). This project is static (no database), but I still used appropriate form input types (email) and required fields to reduce obvious invalid input. In a real deployment, the form should be processed server-side with validation and secure storage.

Copyright issue

Website images must be copyright-safe. I would use my own photos or images from free-to-use sources (e.g. Unsplash/Pexels) and keep a small credit list. For submission, I would also avoid copying text from other websites and write content in my own words.

Censorship / content-appropriateness issue

Public websites should avoid graphic or distressing content (for example, severe injury photos) because it can be upsetting, especially for children. For this website, I kept the gallery and pet images positive and family-friendly, and used neutral, respectful language throughout.
